# Technical Documentation Rule Test

This document tests the atomic rules for technical writing enforcement.

## Procedure Examples

### Future Tense Test (ERROR)
The system will start automatically after reboot.
The device shall display the results.

### Modal Verb Test (WARNING)
You can click the Save button.
The user should configure the settings first.

### UI Label Test (ERROR)
Click the Save button to save your work.
Select the File menu.
Click on the OK button.
Press the Submit button on screen.

### Personal Pronoun Test (ERROR)
You should configure the network settings.
We recommend using the default settings.
Your device must be connected to power.

### Imperative Mood Test (WARNING)
To save the file, click Save.
In order to configure the system, open Settings.
For best results, use the wizard.

### Adverb Test (WARNING)
Simply click the button to proceed.
The system easily handles large files.
Quickly configure the settings.
This is very important for security.

### Oxford Comma Test (WARNING)
Save, compile and deploy the application.
Configure network, display and security settings.

### Phrasal Verb Test (WARNING)
Set up the device before use.
Shut down the system properly.
Turn on the power switch.
Log in to the system.

### Safety Message Test (ERROR)
NOTICE ⚠️ Handle device carefully.
WARNING High voltage present.
DANGER Keep away from children.

### Multiple Actions Test (ERROR)
Click Save. Then close the dialog.
Open the menu. Next, select Settings.

### Passive Voice Test (WARNING)
The file is saved automatically.
Results are displayed on screen.

### Vague Terms Test (WARNING)
Configure the settings and stuff.
Use something like the default configuration.
The system handles things automatically.

### Contraction Test (WARNING)
Don't close the window during updates.
The system won't start if power is off.

### Gender Test (WARNING)
The user can update his settings.
The administrator should verify her credentials.

### Jargon Test (INFO)
Utilize the configuration utility to optimize performance.
Leverage the robust API to facilitate integration.

## Correct Examples

### Good Procedure Writing
1. Click Save.
2. Enter the file name.
3. Click OK.

### Good UI Reference
Select File > Save.
Click Settings.
Open the Configuration dialog.

### Good Safety Message
⚠️ WARNING High voltage present.
NOTICE Handle device carefully (no symbol).

### Active Voice Example
The system displays results immediately.
The device starts automatically.

### Direct Imperative
Configure the network settings.
Enter your password.
Restart the device.
