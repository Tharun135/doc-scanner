# Example Good Technical Documentation

## Clear Instructions
Open the configuration panel. Select the target device from the dropdown menu. Enter the IP address. Click Apply to save changes.

## Active Voice Examples
The system monitors network traffic continuously.
The application validates input before processing.
Users configure settings through the web interface.
The connector synchronizes data every 5 minutes.

## Concise Writing
Navigate to File > Export. Select the L5X format. Choose your output folder. Click Transform to begin.

## Procedure Example
To configure the device:
1. Log in to the management console
2. Select the device from the list
3. Update the configuration parameters
4. Save your changes
5. Restart the device

## Avoiding Passive Voice
Instead of: "The program is executed by the system"
Write: "The system executes the program"

Instead of: "The data is stored in the database"
Write: "The database stores the data"

Instead of: "The file can be accessed by users"
Write: "Users can access the file"

## Breaking Long Sentences
Instead of: "Open the Studio 5000 application from Rockwell Automation and navigate to the file menu and then select the save as option and choose the L5X format"

Write: "Open the Studio 5000 application from Rockwell Automation. Navigate to the file menu. Select Save As. Choose the L5X format."

## Removing Unnecessary Words
Instead of: "Click on the button previously mentioned"
Write: "Click the button"

Instead of: "It is important to note that you should"
Write: "You should"

Instead of: "In order to configure the settings"
Write: "To configure the settings"
